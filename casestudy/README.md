# Engineering Case Studies

Real-world engineering challenges and solutions from production systems.

## Purpose

Learn from how companies solved specific problems:
- Production incidents and their resolutions
- Architecture decisions and migrations
- Scaling challenges and solutions
- System failures and recovery strategies

Each case study documents the problem, context, solution, and outcomes from actual engineering teams.

## Structure

Cases organized by domain:

- **Database** - Storage, scaling, migrations, outages
- **Distributed Systems** - Consensus, partitioning, failure handling
- **Performance** - Bottlenecks, optimizations, profiling
- **Reliability** - Incidents, postmortems, recovery
- **Infrastructure** - Deployments, networking, cloud migrations

## Format

Each case study includes:

- **Company & Context** - Who, when, scale
- **Problem** - What went wrong or needed solving
- **Investigation** - How the issue was diagnosed
- **Solution** - What was implemented
- **Outcome** - Results and lessons learned
- **Related Patterns** - Links to system-design patterns

## Sources

Collected from:
- Engineering blogs
- Conference talks
- Postmortem reports
- Technical documentation

All cases cite original sources.
